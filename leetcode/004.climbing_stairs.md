# 4. 爬楼梯（Climbing Stairs）

## 一、问题描述

假设你正在爬楼梯，需要 n 阶才能到达楼顶。每次你可以爬 1 个或 2 个台阶，问有多少种不同的方法可以爬到楼顶？

示例

输入：n = 2  
输出：2  
解释：有两种方法可以爬到楼顶。  
 1 阶 + 1 阶  
 2 阶  

输入：n = 3  
输出：3  
解释：有三种方法可以爬到楼顶。  
1 阶 + 1 阶 + 1 阶  
1 阶 + 2 阶  
2 阶 + 1 阶  

## 二、解题思路

核心思想：*动态规划*

爬楼梯问题的本质是斐波那契数列的应用，核心逻辑是「状态转移」，通过拆解问题避免重复计算，提升解题效率。

- 状态定义：设 dp[i] 表示爬到第 i 阶楼梯的总方法数。
- 状态转移方程：要到达第 i 阶，最后一步只有两种可能——从第 i-1 阶爬 1 步，或从第 i-2 阶爬 2 步，因此：
  
  dp[i] = dp[i-1] + dp[i-2]

- 初始条件：
  
  dp[1] = 1（1 阶楼梯只有 1 种爬法）  
  dp[2] = 2（2 阶楼梯有 2 种爬法：1+1、2）

## 三、代码实现

```python
def climbStairs(n: int) -> int:
    """
    爬楼梯问题 - 动态规划解法
    :param n: 楼梯总阶数（约束：1 ≤ n ≤ 45）
    :return: 爬到楼顶的不同方法数
    """
    # 处理边界情况：n=1或n=2时，直接返回对应方法数
    if n <= 2:
        return n
    
    # 初始化dp数组：dp[0]无实际意义，仅用于对齐索引（使dp[1]对应1阶，dp[2]对应2阶）
    dp = [0] * (n + 1)
    dp[1] = 1  # 1阶楼梯：1种方法
    dp[2] = 2  # 2阶楼梯：2种方法
    
    # 从第3阶开始，递推计算每一阶的方法数
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]
```
## 四、补充说明

### 什么是动态规划？
动态规划是一种通过把原问题分解为相对简单的子问题，并存储子问题的解来避免重复计算的算法设计方法。动态规划的核心思想是**状态转移**，我们通过定义**状态**（即子问题的描述），并找出状态之间的递推关系，然后从最小的状态开始，逐步计算出所有状态的值，最终得到原问题的答案。

### 解题步骤
1. **定义状态**  
   用一个数组来表示子问题的解。例如在爬楼梯问题中，我们定义 `dp[i]` 表示爬到第 `i` 阶楼梯的不同方法数。

2. **确定状态转移方程**  
   思考如何从较小的子问题的解推导出当前子问题的解，这通常需要分析最后一步或最后的选择。对于爬楼梯，要到达第 `i` 阶，最后一步要么是从 `i-1` 阶跨 1 步，要么是从 `i-2` 阶跨 2 步。因此：  
   `dp[i] = dp[i-1] + dp[i-2]`

3. **初始化边界条件**  
   最小的子问题（无法分解）需要直接给出值，作为递推的起点。例如：  
   `dp[1] = 1`，`dp[2] = 2`。
