# 33.最大子数组之和

## 题目描述

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组是数组中的一个连续部分。

 **示例 1：**

输入：nums = [-2,1,-3,4,-1,2,1,-5,4]

输出：6

解释：连续子数组 [4,-1,2,1] 的和最大，为 6 。

**示例 2：**

输入：nums = [1]

输出：1

**示例 3：**

输入：nums = [5,4,-1,7,8]

输出：23

## 解题思路

本题考查的是动态规划思想。从头开始依次求和，对于当前的元素，如果大于之前的子数组和就舍弃前面的从当前开始算，否则继续相加，每次都要比较并更新最大的和。

如何区分贪心算法和动态规划呢？

题目问什么？

├── "能否..." / "最少次数..." / "最大收益..." 

│   └── 看是否有**最优子结构**

│       ├── 当前选择影响后续所有状态 → 动态规划

│       └── 当前选择只影响当下，不回头修改 → 贪心

│

└── "所有方案..." / "方案数..." / "具体路径..."

    └── 动态规划（或回溯）
