# 24.电话号码的字母组合

## 题目描述

给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。答案可以按 任意顺序 返回。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

**示例 1：**

输入：digits = "23"

输出：["ad","ae","af","bd","be","bf","cd","ce","cf"]

**示例 2：**

输入：digits = "2"

输出：["a","b","c"]

## 解题思路

首先使用哈希表存储每个数字对应的所有可能的字母，然后进行**回溯操作**。

回溯过程中维护一个字符串，表示已有的字母排列（如果未遍历完电话号码的所有数字，则已有的字母排列是不完整的）。该字符串初始为空。每次取电话号码的一位数字，从哈希表中获得该数字对应的所有可能的字母，并将其中的一个字母插入到已有的字母排列后面，然后继续处理电话号码的后一位数字，直到处理完电话号码中的所有数字，即得到一个完整的字母排列。然后进行**回退操作**，遍历其余的字母排列。

回溯算法用于寻找所有的可行解，如果发现一个解不可行，则会舍弃不可行的解。在这道题中，由于每个数字对应的每个字母都可能进入字母组合，因此不存在不可行的解，直接穷举所有的解即可。

## 代码示例
```python
def letterCombinations(self, digits):
        """
        :type digits: str
        :rtype: List[str]
        """
        if not digits:
            return list()
        phoneMap={
             "2": "abc",
            "3": "def",
            "4": "ghi",
            "5": "jkl",
            "6": "mno",
            "7": "pqrs",
            "8": "tuv",
            "9": "wxyz",
        }
        
        def backtrack(index):
            if len(digits)==index:
                combinations.append("".join(combination))
            else :
                digit=digits[index]
                for letter in phoneMap[digit]:
                    combination.append(letter)
                    backtrack(index+1)
                    combination.pop()
        combination=list()
        combinations=list()
        backtrack(0)#digits'index
        return combinations
```

## 代码说明

为什么要用combinations.pop()?

这个是回溯操作的核心机制。

开始: combination = []

backtrack(0): 处理数字 "2" (abc)
 
  ├─ 选 'a' → combination = ['a']
  
  │   └─ backtrack(1): 处理数字 "3" (def)
  
  │       ├─ 选 'd' → combination = ['a','d'] → 加入结果 "ad"
  
  │       │   └─ 返回，执行 pop() → ['a']  ← ⚠️ 关键！撤销'd'
  
  │       ├─ 选 'e' → combination = ['a','e'] → 加入结果 "ae"
  
  │       │   └─ 返回，执行 pop() → ['a']  ← ⚠️ 撤销'e'
  
  │       └─ 选 'f' → combination = ['a','f'] → 加入结果 "af"
  
  │           └─ 返回，执行 pop() → ['a']  ← ⚠️ 撤销'f'
  
  │   └─ 回到上一层，执行 pop() → []  ← ⚠️ 撤销'a'
  
  │
  
  ├─ 选 'b' → combination = ['b']  (重新开始)
  
  │   └─ backtrack(1): 同理产生 "bd", "be", "bf"
  
  │   └─ pop() → []
  
  │
  
  └─ 选 'c' → combination = ['c']  (再次重新开始)
  
      └─ backtrack(1): 同理产生 "cd", "ce", "cf"
      
      └─ pop() → []

  pop()操作可以撤销这一步，返回上一次的状态，以便于进行其他可能情况的尝试。

  为什么没有return？

  递归的返回不需要显式 return！当函数执行完毕（遇到 } 或最后一行），会自动返回到调用它的地方。
